# Backend Role-Gated API

Backend API for team-scoped operations with Firebase Auth verification and role checks.

## What it solves

- Only `leader` or `collaborator` can:
  - add team members
  - create team rewards
  - create achievements
  - update team info
- Team creator becomes leader automatically.
- User rewards can be created only by system endpoints.

## Base URL

- Default local base URL: `http://localhost:8080/api/v1`
- Health checks:
  - `GET /health`
  - `GET /api/v1/health`

## Endpoints

- `POST /api/v1/teams`
- `PATCH /api/v1/teams/:teamId`
- `POST /api/v1/teams/:teamId/members` (invite, leader/collaborator)
- `PATCH /api/v1/teams/:teamId/members/:userUid/role` (role change, leader only)
- `POST /api/v1/teams/:teamId/rewards`
- `POST /api/v1/teams/:teamId/achievements`
- `POST /api/v1/internal/users/:userUid/rewards` (requires `x-system-key`)

## Setup

1. Copy `.env.example` to `.env`.
2. Set `DATABASE_URL`, `SYSTEM_API_KEY`, and service account credentials.
3. Configure frontend origin in `ALLOWED_ORIGINS`.
4. Install and run.

```bash
cd backend
npm install
npm run dev
```

## Auth

- Client endpoints require `Authorization: Bearer <Firebase ID token>`.
- Internal endpoint requires `x-system-key`.

## Notes

- SQL targets tables generated by your Data Connect schema (`team`, `team_membership`, `team_reward`, `achievement`, `user_reward`).
- Use this API from frontend instead of exposing sensitive Data Connect mutations to clients.
- This folder is ready to move to a standalone repository with minimal changes.
